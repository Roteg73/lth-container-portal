# DESIGN_GUIDE.md — Designriktning

## Känsla

LTH Container Portal ska upplevas som:

- industriell
- modern
- premium
- tekniskt trovärdig
- ren och tydlig
- byggd för B2B och industrikunder

Riktning: mer Volvo CE / Cramo / Ramirent än standard-WordPress.

## Visuella principer

- Stora bilder.
- Tydliga rubriker.
- Kortare textblock.
- Konkreta nyttor.
- Kraftig typografi.
- Luft mellan sektioner.
- Mörka industriella ytor kan användas.
- Accentfärg används sparsamt.
- Knappar ska vara tydliga och säljdrivande.
- Innehållet ska fungera i kundmöte.

## Layout

Föreslagna sektioner för publik sida:

1. Hero
2. Kort värdeerbjudande
3. Typiska användningsområden
4. Varför containerlösningen
5. Process från behov till leverans
6. Kontakt/CTA

Föreslagna sektioner för låst portal:

1. Kundanpassad hero
2. Översikt
3. Lösningsmoduler
4. Bild/film
5. Tekniska dokument
6. Nästa steg/kontakt
7. Frågor och svar

## Responsivitet

Designen ska fungera för:

- desktop i säljmöte
- surfplatta
- mobil

Undvik komponenter som endast fungerar med hover.

## Textprincip

Skriv inte romaner i UI.

Använd:

- korta rubriker
- kort ingress
- punktlistor
- tydliga CTA
- visuella moduler

## Modulprincip

Presentationens innehåll ska byggas så att samma delar senare kan återanvändas för:

- webbpresentation
- PDF
- PowerPoint
- offertbilagor

Undvik hårdkodad jättesida där allt ligger i en enda mall.

## Tillgänglighet

- Bra kontrast.
- Knappar ska vara klickbara med tydlig fokus.
- Bildtexter/alt där relevant.
- Text ska kunna läsas på mobil.

## Vad som ska undvikas

- standardbloggkänsla
- för mycket brödtext
- plottriga widgets
- alltför många färger
- blinkande effekter
- tunga animationer som gör portalen långsam
- innehåll som avslöjar känslig teknik/pris publikt
