# CODEX_WORKFLOW.md — Arbetsflöde med Codex

## Princip

Codex är utvecklare. ChatGPT/projektägaren tar fram nästa tydliga task. Codex utför bara aktuell task.

## Start av ny task

1. Läs `AGENTS.md`.
2. Läs `CURRENT_TASK.md`.
3. Läs relevanta projektdokument.
4. Kontrollera befintlig kod.
5. Utför tasken.
6. Testa.
7. Rapportera.

## Codex ska inte

- hitta på nästa roadmapsteg själv
- bygga flera faser samtidigt
- ändra live
- ändra WordPress core
- lägga känslig data i repo
- hoppa över säkerhetskontroller

## Efter Codex-rapport

Projektägaren granskar:

- om tasken följt omfattning
- om säkerhetskraven är uppfyllda
- om testresultatet är trovärdigt
- om filändringar är rimliga
- om nästa task kan tas fram

## Nästa task

Nästa task tas fram först efter granskning.
